# CookFlow — Продуктове ТЗ для MVP

**Версія:** 1.0  
**Дата:** Квітень 2026  
**Платформа:** Flutter (iOS + Android)  
**Референс:** HTML демо `cookflow-demo-v5.html`

---

## 1. Огляд продукту

### 1.1 Ідея

CookFlow — мобільний додаток для покрокового приготування їжі з інтелектуальним графом залежностей між кроками і таймерами у фоні. Користувач бачить один крок за раз, не відволікається, не пропускає нічого.

### 1.2 Ключовий інсайт

Існуючі рецептні додатки показують список інгредієнтів і текст рецепту — як книга. CookFlow веде користувача як GPS: наступний крок доступний тільки коли виконані попередні або завершився таймер. Паралельні процеси (бульйон вариться — ріжемо овочі) керуються автоматично.

### 1.3 Цільова аудиторія

- Люди що готують вдома 2–5 разів на тиждень
- Початківці яким потрібна впевненість і структура
- Досвідчені кухарі що хочуть не тримати рецепт в голові

### 1.4 Метрики успіху MVP

- Час до першого завершеного рецепту < 3 хв після встановлення
- % користувачів що дійшли до кроку "Готово" > 60%
- Retention D7 > 25%

## 2. Архітектура продукту

### 2.1 Флоу користувача

```
Головна (каталог)
  → Деталі рецепту (інфо + інгредієнти)
    → Перед готуванням (чекліст інгредієнтів | огляд кроків)
      → Готування (граф кроків, один за раз)
        → Готово 🎉
```

### 2.2 Екрани MVP

| Екран | ID | Опис |
|---|---|---|
| Головна | `home` | Каталог рецептів |
| Деталі рецепту | `detail` | Інфо, інгредієнти, оцінки |
| Перед готуванням | `checklist` | 2 таби: інгредієнти + огляд кроків |
| Готування | `cook` | Покроковий режим |
| Завершено | `done` | Екран успіху з конфеті |
| Список покупок | `shopping` | Відсутні інгредієнти |
| Профіль | `profile` | Статистика користувача |

## 3. Модель даних

### 3.1 Recipe

```dart
class Recipe {
  String id;
  String name;
  String emoji;
  int baseServings;
  int timeMinutes;       // загальний час
  int kcalPerServing;
  String difficulty;     // 'easy' | 'medium' | 'hard'
  String category;       // 'soup' | 'main' | 'breakfast' | 'salad'
  List<Ingredient> ingredients;
  List<Step> steps;      // граф кроків
  double rating;
  int reviewCount;
}
```

### 3.2 Ingredient

```dart
class Ingredient {
  String name;
  double? amount;        // null = "за смаком"
  String unit;           // 'г' | 'шт' | 'ст.л.' | 'ч.л.' | 'пучок' | 'за смаком'
  String category;       // 'meat' | 'vegs' | 'dairy' | 'eggs' | 'other'
}
```

### 3.3 Step — ключова сутність

```dart
class Step {
  int id;
  int weight;            // пріоритет для сортування (менше = раніше)
  List<int> deps;        // id кроків що мають бути done=true
  int? timerDep;         // id кроку чий bg-таймер має завершитись
  String text;           // текст дії
  StepNote? note;        // підказка або попередження
  StepTimer? timer;      // null або таймер
  bool waitTimer;        // true = кнопка "Далі" блокується до кінця таймера
}

class StepNote {
  String type;           // 'tip' | 'warn' | 'pro'
  String text;
}

class StepTimer {
  int minutes;
  String label;
  bool isBackground;     // true = таймер у фоні, можна перейти далі
}
```

### 3.4 Граф залежностей — алгоритм

```
pickNextStep():
  1. Відфільтрувати: !done && deps.every(done) && timerDep завершено
  2. Відсортувати за weight (ascending)
  3. Повернути перший
  
Якщо результат null але є незавершені кроки → екран очікування
Якщо всі виконані → екран завершення
```

### 3.5 Стан кроку під час готування

```dart
class StepState {
  bool done;
  DateTime? readyAt;     // коли стане доступним через timerDep
  TimerState? timerState;
}

class TimerState {
  int secondsRemaining;
  int totalSeconds;
  bool isRunning;
  bool isDone;
}
```

- Кнопка "🍳 Починаємо готувати →" (завжди активна)

### 4.4 Готування (cook) — КЛЮЧОВИЙ ЕКРАН

**Структура екрану (зверху вниз):**

```
[Float bars — активні фонові таймери]
[Top bar: кнопка виходу ✕ | прогрес-бар | N/Total]
[Step Card — займає весь простір]
[Bottom button — одна кнопка]
```

**Float bars (фонові таймери):**
- Вузька смужка зверху для кожного активного bg-таймера
- Показує: назву, час що залишився (великий шрифт), анімована точка
- Максимум 2 одночасно
- При < 1 хв до кінця — підсвічується червоним
- Тап → нічого (тільки індикатор)

**Step Card:**
```
Крок N з Total
[Великий заголовок — текст кроку]
[Note block якщо є — warn/tip/pro]
[Timer block якщо є]
```

**Note block кольори:**
- `warn` — червонуватий фон, "⚠️ текст"
- `tip` — жовтуватий фон, "💡 текст"  
- `pro` — зеленуватий фон, "🏆 текст"

**Timer block:**
```
[Назва таймера]              [ММ:ХХ — великий шрифт]
[⚡ Продовжується у фоні]  (якщо bg)
[====Progress bar====]
```
- Без кнопок всередині блоку — тільки індикатор
- Якщо bg — підпис "⚡ Продовжується у фоні після Далі"
- Остання 15% часу → цифри червоніють + пульсують

**Bottom button — логіка станів:**

| Стан | Текст кнопки | Дія при натисканні |
|---|---|---|
| Нема таймера | "Далі →" | Перейти до наступного кроку |
| Bg таймер, не запущений | "Далі →" | Запустити таймер у фоні + перейти |
| Bg таймер, запущений | "Далі →" | Перейти (таймер продовжується) |
| Блокуючий таймер, не запущений | "Далі →" | Запустити таймер (НЕ переходити) |
| Блокуючий таймер, іде | "⏸ Пауза" | Поставити на паузу |
| Блокуючий таймер, пауза | "▶ Продовжити" | Відновити |
| Блокуючий таймер, завершено | "Далі →" | Перейти |
| Останній крок | "🎉 Готово!" | Перейти на екран завершення |

**Екран очікування (якщо pickNextStep = null але є кроки):**
- Показує активні таймери з відліком
- Текст "Наступний крок з'явиться автоматично"
- Кнопка неактивна "Очікування..."
- Автоматично переходить коли таймер завершується

**Діалог виходу:**
- "Зупинити готування?"
- "Продовжити готувати" | "Так, зупинити"
- При виході — зупиняються всі таймери

### 6.2 State Management

Рекомендую **Riverpod** або **BLoC/Cubit**. Для кожного екрану — окремий провайдер/кубіт.

`CookCubit` — найважливіший:
```dart
class CookState {
  Recipe recipe;
  Map<int, StepState> stepStates;  // стан кожного кроку
  int? currentStepId;
  bool isWaiting;                  // немає доступних кроків
}
```

### 6.3 Таймер сервіс

- Один `Timer.periodic(Duration(seconds: 1))` на весь сеанс готування
- Кожен тік — декрементує всі активні таймери
- При завершенні bg-таймера: оновлює `readyAt` → `pickNextStep()` → якщо є наступний → перейти
- При завершенні blocking-таймера: оновлює стан → кнопка змінюється на "Далі →"
- Push notification при завершенні bg-таймера якщо додаток у фоні

### 6.4 Локальне сховище

MVP — тільки локальні дані:
- Рецепти: hardcoded в коді (JSON файли в assets)
- Список покупок: SharedPreferences або Hive
- Профіль (ім'я, кількість приготованих): SharedPreferences
- Збережені рецепти: SharedPreferences (список id)

### 6.5 Push Notifications

Необхідно для bg-таймерів:
- `flutter_local_notifications`
- При виклику `_launchTimer(bgTimer)` → планувати нотифікацію через N секунд
- Текст: "⏱ [Назва таймера] — час вийшов!"
- При поверненні в додаток — скасувати заплановану нотифікацію

## 8. Сценарії використання (User Stories)

```
Як користувач
Я хочу переглянути каталог рецептів

AC:
- Бачу карточки з назвою, часом, ккал, рейтингом
- Можу тапнути на карточку щоб відкрити деталі
- Можу зберегти в обране
```

### US-2: Перевірка інгредієнтів
```
Як користувач
Я хочу відмітити які інгредієнти є вдома
Щоб розуміти що треба купити

AC:
- Бачу всі інгредієнти з кількостями
- Тап = відмітити "є вдома" (зеленим)
- Кнопка додає відсутні до списку покупок
- Кількості масштабуються з порціями
```

### US-3: Огляд кроків перед стартом
```
Як користувач
Я хочу переглянути всі кроки перед початком
Щоб розуміти що мене чекає

AC:
- Таб "Кроки" показує повний список у правильному порядку
- Бачу які кроки мають таймер і скільки хвилин
- Бачу які таймери фонові
```

### US-4: Покроковий режим готування
```
Як користувач
Я хочу бачити один крок за раз
Щоб не відволікатись і нічого не пропустити

AC:
- Бачу поточний крок великим текстом
- Одна кнопка внизу — завжди зрозуміло що робити
- Підказки і попередження прямо на екрані
- Не треба думати що далі — додаток сам каже
```

### US-5: Фонові таймери
```
Як користувач
Я хочу запустити бульйон і паралельно різати овочі
Щоб не гаяти часу

AC:
- Після "Далі" на фоновому таймері — бачу смужку зверху з відліком
- Наступні кроки доступні одразу
- Push-повідомлення коли таймер завершився
- При поверненні — наступний крок вже показується
```

### US-6: Блокуючий таймер
```
Як користувач
Я запустив смажити цибулю
Я хочу знати коли можна переходити далі

AC:
- Натискаю "Далі →" — таймер запускається
- Кнопка стає "⏸ Пауза" поки іде
- Після завершення — автоматично змінюється на "Далі →"
- Можу поставити на паузу і відновити
```

### US-7: Список покупок
```
Як користувач
Я хочу бачити список того що треба купити
Щоб не забути нічого в магазині

AC:
- Бачу список відсутніх інгредієнтів з кількостями
- Можу відмітити "куплено"
- Список зберігається між сеансами
```

## 10. Рекомендації AI-агенту

### Референс

Вся UI/UX логіка детально реалізована в `cookflow-demo-v5.html`. Використовуй його як:
- Точний референс для кольорів, відступів, компонентів
- Джерело даних рецептів (R_BORSCHT, R_OMELETTE, STEPS_BORSCHT, STEPS_OMELETTE)
- Логіку алгоритму `pickNextStep()` → портуй на Dart 1:1

### Пріоритети реалізації

1. **Cook screen** — серце продукту, починай з нього
2. **Граф кроків** — `pickNextStep()`, `StepState`, таймери
3. **Checklist** — простий, але важливий для конверсії
4. **Detail screen** — масштабування порцій
5. **Home screen** — каталог, останнє

### Критичні деталі

- Таймери: один `Timer.periodic` на весь сеанс (не окремий на кожен крок)
- Float bars: показувати тільки `isBackground = true` таймери що ще йдуть
- `pickNextStep()` викликається після кожного "Далі" і після кожного завершення таймера
- `readyAt` — DateTime в реальному часі, не демо-швидкість
- Bottom button `onclick` змінюється динамічно залежно від стану (не disabled/enabled)

### Що НЕ копіювати з HTML демо

- `DEMO_SPEED = 6` — прискорення таймерів. В реальному додатку `minutes × 60` реальних секунд
- Заглушки locked карточок ("Скоро") — замінити на реальні рецепти або прибрати
- `cookedN = 12` — hardcoded лічильник, замінити на реальний стан
